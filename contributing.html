# Contributing to AeroPax Live Thank you for your interest in contributing to AeroPax Live! We welcome contributions from the community to help make this flight tracking app even better. ## How to Contribute ### Reporting Bugs If you find a bug, please create an issue with the following information: - **Title:** A clear and descriptive title - **Description:** A detailed description of the problem - **Steps to Reproduce:** Step-by-step instructions to reproduce the issue - **Expected Behavior:** What you expected to happen - **Actual Behavior:** What actually happened - **Environment:** - iOS version - Device model - App version - **Screenshots:** If applicable, add screenshots or screen recordings ### Suggesting Enhancements We love new ideas! To suggest an enhancement: - Check existing issues to see if it's already been suggested - Create a new issue with the "enhancement" label - Clearly describe the feature and its benefits - Provide examples or mockups if possible ### Pull Requests We accept pull requests for bug fixes and approved features. Before submitting: 1. **Fork the repository** and create your branch from `main` 2. **Follow Swift conventions:** - Use Swift 6 features where appropriate - Follow Apple's Swift API Design Guidelines - Use Swift Concurrency (async/await) for asynchronous operations - Prefer SwiftUI over UIKit for new UI components 3. **Write clean code:** - Add comments for complex logic - Use meaningful variable and function names - Keep functions focused and concise 4. **Test your changes:** - Ensure the app builds without warnings - Test on multiple iOS versions if possible - Verify that existing functionality still works 5. **Update documentation** if you're changing behavior 6. **Submit your pull request** with: - A clear title and description - Reference to any related issues - Screenshots of UI changes (if applicable) ## Development Setup ### Prerequisites - Xcode 15.0 or later - iOS 17.0 SDK or later - macOS 14.0 or later - Swift 6.0 or later ### Getting Started ```bash # Clone the repository git clone https://github.com/yourusername/aeropax-live.git # Open in Xcode cd aeropax-live open AeroPax_Live.xcodeproj ``` ### Project Structure - `AeroPax_LiveApp.swift` - Main app entry point - `ContentView.swift` - Main content view - `FlightHistory.swift` - Flight record model - `FlightHistoryView.swift` - Flight history interface - `ArrivalAlertView.swift` - Arrival alert functionality ## Code Style Guidelines ### Swift Style - Use 4 spaces for indentation - Follow Apple's Swift naming conventions - Use explicit `self` only when required - Prefer `let` over `var` when possible - Use trailing closures when appropriate ### SwiftUI Best Practices - Break complex views into smaller, reusable components - Use `@State` and `@Binding` appropriately - Leverage SwiftData for persistence - Use Swift Concurrency for async operations ### Example Code Style ```swift import SwiftUI import SwiftData struct FlightDetailView: View { @State private var isLoading = false let flight: FlightRecord var body: some View { VStack(alignment: .leading, spacing: 16) { Text(flight.flightNumber) .font(.title) if isLoading { ProgressView() } else { flightDetails } } .padding() } private var flightDetails: some View { VStack { // Flight details implementation } } } ``` ## Testing ### Manual Testing Checklist - [ ] App launches without crashes - [ ] Location permissions work correctly - [ ] Flight tracking records data accurately - [ ] Flight history displays correctly - [ ] Data persists after app restart - [ ] UI looks good on different screen sizes - [ ] Dark mode support works properly ### Automated Testing If you're adding new features, consider adding tests using Swift Testing: ```swift import Testing @Test("Flight record saves correctly") func testFlightRecordSaving() async throws { let record = FlightRecord(/* parameters */) // Test implementation #expect(record.isValid) } ``` ## Documentation When adding new features: - Update relevant `.md` files - Add inline code comments for complex logic - Update the README if necessary - Document any new dependencies ## Communication - **Email:** admin@av8r.com.au for direct inquiries - **GitHub Issues:** For bug reports and feature requests - **Pull Requests:** For code contributions ## Code of Conduct ### Our Standards - Be respectful and inclusive - Welcome newcomers and help them learn - Focus on constructive feedback - Respect different viewpoints and experiences ### Unacceptable Behavior - Harassment or discriminatory language - Trolling or insulting comments - Publishing others' private information - Other conduct that's inappropriate in a professional setting ## Recognition Contributors will be recognized in our release notes and may be listed in a CONTRIBUTORS file. ## Questions?
